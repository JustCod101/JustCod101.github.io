防火长城（GFW）技术鉴赏与深度剖析

防火长城（Great Firewall, 简称 GFW）是中国国家防火墙的非正式名称，它是一套复杂、多维度的互联网审查和分析系统。其核心目的不仅是物理拦截，更是对跨境流量的深度感知与控制。

1. 防火长城的拓扑结构：旁路部署

GFW 并非传统意义上的“串行防火墙”。为了不影响庞大的跨境流量带宽性能，GFW 主要采用 旁路（Out-of-band） 部署。

通过分光器或镜像端口，GFW 能够实时获取骨干网（AS4134, AS4837 等）出口路由的流量副本。当它识别出违规流量时，会异步地向连接双方发送干扰信号（如 TCP RST），从而达到阻断连接的目的。

2. 核心审查机制

2.1 DNS 污染与劫持 (DNS Poisoning)

这是 GFW 最古老且最常用的手段。

  • 原理：由于 DNS 查询通常通过 UDP 53 端口以明文形式进行，GFW 的 IDS（入侵检测系统）会嗅探包含黑名单域名的查询请求。
  • 干扰：GFW 会抢在真实的 DNS 服务器之前，返回一个虚假的 IP 地址（通常是无效的或被封锁的 IP）。由于 UDP 是无连接的，查询者会优先接受第一个到达的（虚假的）回复。

2.2 IP 地址与传输层端口封锁

  • 早期：使用访问控制列表（ACL）静态封锁目标 IP。
  • 现代：采用 BGP 路由黑洞（Blackholing） 技术。通过向出口路由器注入错误的 BGP 路由信息，使发往受控 IP 的流量被引导至黑洞设备或被丢弃。这种方式效率极高，且难以通过简单的手段绕过。

2.3 TCP 连接重置 (TCP Reset Attack)

  • 原理：GFW 监测到 HTTP GET 请求或其他明文协议中包含关键词（Keyword Filtering）时，会伪造带有 RST 标志位的 TCP 报文发送给通信双方。
  • 后果：通信双方的操作系统会误以为对方主动关闭了连接，从而立即中断当前的会话。

2.4 深度包检测技术 (DPI)

DPI (Deep Packet Inspection) 是 GFW 的“大脑”。它不仅检查报文头，更会解析应用层负载。

  • 关键词过滤：基于特定敏感词的正则匹配。
  • 协议识别：GFW 能够识别出未经过伪装的 VPN 协议（如 PPTP, OpenVPN）或加密代理流量（如早期的 Shadowsocks）。
  • 证书分析：在 TLS 1.2 握手过程中，由于 ServerHello 阶段的站点证书是明文传输的，GFW 可以通过读取证书中的 SNI（Server Name Indication）或通用名称（CN）来判定访问的域名。

3. 科学上网协议的博弈

针对 GFW 的封锁，技术社区不断开发出新的协议进行对抗。

3.1 VPN (Virtual Private Network)

VPN 最初设计用于远程接入企业私有网络。

  • 优点：强加密，隧道化所有流量。
  • 局限性：特征明显。传统的 PPTP/L2TP 协议极易被 DPI 识别。在 GFW 环境下，由于 VPN 协议缺乏“流量混淆”，往往会导致服务器 IP 迅速被封锁。

3.2 代理协议（Shadowsocks, V2Ray, Trojan）

这些协议的核心思想是 “混淆”与“轻量”。

  • Shadowsocks (SS)：由 clowwindy 开发，采用 Socks5 代理与自定义加密，旨在模拟普通的 TCP 流量。
  • VMess / VLESS (V2Ray)：更强大的协议族，支持多种传输方式（WebSocket, gRPC, HTTP/2），并引入了 Mux（多路复用） 提高性能。
  • Trojan：放弃了自定义加密特征，转而直接模仿最普遍的 HTTPS 流量。它将数据封装在标准的 TLS 隧道中，使得流量在外部看来与正常的网页访问无异。

3.3 后 TLS 时代：ECH 与加密 SNI

随着 TLS 1.3 的普及，越来越多的握手信息被加密。

  • ECH (Encrypted Client Hello)：作为 TLS 的扩展，它加密了整个 Client Hello 报文，包括 SNI。这将使得 GFW 无法再通过被动侦听获取用户访问的目标主机名。

4. 结论与反思

GFW 与反审查技术之间的较量是一场长期的“猫鼠游戏”。从最初的简单拦截到现在的机器学习识别和主动探测，GFW 的技术边界在不断扩展。理解其原理，不仅是为了“科学上网”，更是为了深刻认识当代互联网的基础设施与信息流动的主权博弈。

信息收集：渗透测试的第一步

在网络安全领域，“信息收集（Reconnaissance）” 是渗透测试中耗时最长、也是最关键的阶段。它决定了攻击面的广度以及后续漏洞挖掘的成功率。

本文将介绍从被动侦察到主动扫描的核心技术与工具。

1. Google Hacking (Dorks)

Google 搜索引擎是世界上最大的信息数据库之一。通过特定的 Google Dorks 语法，我们可以发现泄露的敏感信息、后台登录口及过时的服务。

核心语法

  • site:target.com：限制搜索特定的域名。
  • filetype:pdf/xls/docx：查找特定的文档（常用于寻找公司敏感报表）。
  • intitle:"Index of"：寻找开启了目录遍历的服务器。
  • inurl:admin/login：寻找后台管理系统的入口。
  • intext:"password"：在网页正文中搜索关键字。

2. 空间搜索引擎：Shodan 与 Censys

Shodan 被称为“黑客界的 Google”，它不爬取网页，而是通过扫描全球的 IP 和端口，识别设备的服务指纹。

Shodan 常用语法

  • net:1.2.3.4/24：搜索特定子网内的设备。
  • port:3389：查找开启了 RDP（远程桌面）的设备。
  • os:"Linux"：指定操作系统。
  • product:"Apache httpd"：指定服务名称。
  • country:CN：指定国家代码（中国）。

3. Nmap：网络映射器的艺术

Nmap 是内网横向渗透和外网资产扫描的必备神器。

扫描策略

  • 隐蔽扫描 (Stealth Scan)：-sS 使用 SYN 扫描，不建立完整 TCP 连接，避免被简单防火墙记录。
  • 服务版本探测：-sV 能够识别端口后面运行的具体版本（如 Apache 2.4.41）。
  • 操作系统指纹：-O 通过 TCP/IP 协议栈的行为推测 OS 类型。
  • 全能模式：-A 包含 OS 检测、版本检测、脚本扫描和路由追踪。

Nmap 脚本引擎 (NSE)

NSE 允许用户编写脚本来自动化漏洞检测。

4. 目录指纹与敏感路径探测

Web 容器的默认路径或遗留的 .git、.env 文件往往是突破口。

  • Dirsearch / ffuf：高效的路径爆破工具。
  • Wappalyzer / WhatWeb：识别网站的技术栈（前端框架、数据库、Web 容器）。

5. 子域名收集：扩展攻击面

对于大型企业，子域名往往包含未被修补的测试系统或备份服务器。

  • Subfinder：利用 API 进行多源子域名收集。
  • Amass：目前最强大的开源域名资产发现工具。

6. 总结：信息收集“三板斧”策略

在面对一个目标时，通常遵循以下流程：

  1. Passive Recon (被动)：通过 Google Dorks, Shodan, WHOIS, DNS 记录（dig, nslookup）获取外围信息，不与目标服务器直接接触。
  2. Asset Discovery (资产发现)：通过 Subfinder 或 Amass 获取所有子域名，确定 IP 范围。
  3. Active Scanning (主动扫描)：针对发现的资产，使用 Nmap 确认存活端口及服务版本，并利用 ffuf 探测敏感目录。

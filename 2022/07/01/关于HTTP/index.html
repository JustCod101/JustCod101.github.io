关于HTTP

HTTP status code

200 : 客户端请求成功

302：重定向

404：请求资源不存在

400：客户端请求有语法错误，不能被服务器所理解

401：请求未经授权

403：服务器收到请求，但是拒绝提供服务

500：服务器内部错误

503：服务器当前不能处理客户端的请求，一段时间后恢复正常

HTTP Header

Request Header

Host

被请求的主机和端口号

User-Agent

客户端的操作系统、浏览器和其他属性

Referer

当前访问URL的上一个URL

Cookie

一段用来表示请求者身份的文本

Range

可以请求实体的部分内容，多线程下载一定会用到此请求头

x-forward-for

代表请求端的IP，可以有多个，中间用逗号隔开

Accept

用于指定客户端接收哪些MIME(Multipurpose Internet Mail Extensions)类型的信息。如Accept : text/html，表示客户端希望接受到HTML文本

Accept-Charset

指定客户端接收的字符集

Response Header

Server

服务器所使用的Web服务器名称，如Server:Apache/1.3.6(Unix)

Set-Cookie

向客户端设置Cookie，通过查看此头，可以看到服务器向客户端发送的Cookie信息

Last-Modified

服务器通过这个头告诉游览器，资源的最后修改时间

Location

服务器通过这个头告诉游览器访问哪个页面

Refresh

服务器通过Refresh头告诉游览器定时刷新游览器

Normal Header

Date

消息产生的日期和时间

Connection

允许发送指定连接的选项

Cache-Control

用于指定缓存指令，缓存指令是单向的，且是独立的

Entity Header

定义了关于实体正文和请求所标识的资源的元信息。

Content-Type

用于向接收方指示实体的介质类型

Content-Encoding

用作媒体的修饰符，其值指示了已经被应用到实体正文的附加内容的编码，因而获得Content-Type报头域中所引用的媒体类型，必须采用相应的编码机制

Content-Length

实体报头用于指明实体正文的长度，以字节方式存储的十进制数字来表示

Last-Modified

最后修改时间

高性能多线程端口扫描器：设计与实现

端口扫描（Port Scanning）是网络安全审计中最基础的技术之一。通过探测目标主机开放的 TCP 端口，我们可以识别其运行的服务，进而评估潜在的攻击面。

本文将展示如何使用 C 语言、Socket 编程及 POSIX 线程（pthreads）构建一个高并发的端口扫描器。

1. 技术背景：TCP 三次握手

本扫描器采用 TCP Connect 扫描（Full Connect Scan）。

  • 原理：程序调用操作系统原生的 connect() 函数。
  • 过程：扫描器尝试与目标端口建立完整的三次握手（SYN -> SYN-ACK -> ACK）。
  • 结果判断：
    • connect() 返回 0：端口 Open。
    • connect() 返回 -1 且 errno 为 ECONNREFUSED：端口 Closed。
    • connect() 超时或无响应：端口可能被防火墙 Filtered。

2. 核心代码实现

相较于单线程逐个扫描，多线程能极大提高探测效率。

3. 性能优化建议 (Masterpiece Section)

  1. 线程池 (Thread Pool)：目前的实现是为每个端口创建一个线程，这在扫描大范围端口（如 1-65535）时会消耗大量系统资源。生产环境下应使用线程池来复用线程。
  2. 非阻塞 I/O (Non-blocking I/O)：通过将 Socket 设置为 O_NONBLOCK 并配合 select() 或 epoll()，可以在单个线程中同时处理成千上万个连接，这是 nmap 等专业工具的核心原理。
  3. SYN 扫描 (Half-open Scan)：通过原始套接字（Raw Sockets）构造报文，只发送 SYN 而不建立完整连接。这种扫描更隐蔽且速度更快，但需要 root 权限。

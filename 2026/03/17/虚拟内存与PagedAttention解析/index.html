跨越半个世纪的共鸣：从 OS 虚拟内存到大模型 PagedAttention 的架构美学

计算机科学中有一句名言：“计算机科学领域的任何问题都可以通过增加一个间接的中间层来解决。”

如果我们在操作系统的演进史和大模型（LLM）推理引擎的发展史中寻找这句名言的最佳注脚，那一定是“分页（Paging）”思想。跨越了近半个世纪，当初为了解决物理内存碎片化而诞生的 OS 分页技术，在 2023 年被 vLLM 团队巧妙地借鉴，化身为解决大模型推理 KV Cache 显存瓶颈的神器——PagedAttention。

本文将带你打通底层 OS 与前沿 AI 的任督二脉，领略这一脉相承的架构美学。

1. 痛点起源：内存碎片与“吃大锅饭”的悲剧

1.1 OS 的远古时代：连续内存分配

在早期的操作系统中，程序运行必须在物理内存中占据一块连续的空间。
假设系统有 100MB 物理内存：

  • 进程 A 占了 0-30MB。
  • 进程 B 占了 30-50MB。
  • 进程 A 运行结束，释放了 0-30MB。
    此时，如果来了一个需要 40MB 的进程 C，系统会报错：内存不足（Out of Memory）。
    为什么？因为虽然物理内存还剩 80MB（30MB + 50MB），但它们是不连续的“碎片”。为了解决这个问题，OS 不得不频繁进行内存紧缩（把进程 B 挪到前面去），但这极其耗时。

1.2 LLM 推理的痛点：KV Cache 的碎片化

历史总是惊人的相似。当我们用 HuggingFace 默认的代码（如 Transformers 库）进行大模型批量推理时，遇到了极其类似的问题。
在 Transformer 架构中，为了避免重复计算，我们会把历史上计算过的 Key 和 Value 张量缓存进 GPU 显存，这就是 KV Cache。
传统框架在为每个 Request 分配 KV Cache 时，是预先分配一块连续的极大显存（假设按 max_sequence_length=2048 分配）。
但实际上，很多句子的长度根本到不了 2048。这就导致了：

  1. 内部碎片：句子只用了 500 个 Token 的空间，剩下 1548 个 Token 的显存被永久白白占着。
  2. 外部碎片：不同请求的长度不同，导致显存中出现大量无法被复用的空隙。
    据统计，在传统框架中，只有 20%-40% 的 KV Cache 显存被真正用于存储状态，其余 60%-80% 全被碎片浪费了。 这就是为什么大模型推理那么费显卡的核心原因。

2. 操作系统的救赎：虚拟内存与分页管理 (Paging)

为了解决碎片问题，现代操作系统引入了虚拟内存（Virtual Memory）和分页（Paging）机制。

2.1 理念拆解

  1. 切碎物理内存：不再要求物理内存是一整块。OS 把物理内存切成一个个大小相等的物理块，称为 页框（Page Frame），通常是 4KB。
  2. 切碎逻辑内存：程序的逻辑地址空间也被切成同等大小的 页（Page）。
  3. 页表映射（Page Table）：OS 为每个进程维护一张“映射表”。程序眼中的连续逻辑地址，通过页表，被映射到物理内存中东一块、西一块的离散页框里。

2.2 降维打击

  • 消灭外部碎片：由于物理内存全被切成了标准的 4KB 小块，只要有空闲块就能分给程序，再也不存在“空间够但不连续”的外部碎片问题。
  • 懒加载（Page Fault）：程序启动时，OS 根本不给它分满物理内存，只有当程序真正访问到某页时，才触发缺页中断去分配，极大提高了内存利用率。

3. AI 的文艺复兴：PagedAttention 的诞生

2023 年，伯克利团队推出了轰动 AI 界的推理引擎 vLLM，其核心创新就是 PagedAttention算法。
它的设计图纸，几乎就是把 OS 的分页管理机制原封不动地搬到了 GPU 显存和注意力机制上。

3.1 Paged KV Cache

像 OS 切分物理内存一样，PagedAttention 将连续的 KV Cache 张量，切分成了一个个固定大小的 Block（块）。

  • 每个 Block 可以存储固定数量（比如 16 个或 32 个）Token 的 KV 向量。
  • 这些 Block 在 GPU 的物理显存中是非连续的、散乱分布的。

3.2 Block Table（AI 版的页表）

既然物理显存是散乱的，Transformer 的 Attention 计算怎么知道该去哪找历史 Token 呢？
vLLM 为每个 Request（对应 OS 中的进程）维护了一张 Block Table（块表）。

  • 逻辑上，这段话的 Token 是一维连续的。
  • 物理上，这段话的前 16 个 Token 存在物理块 101，接着的 16 个 Token 存在物理块 54。
  • 当进行 Attention 矩阵乘法时，CUDA Kernel 会查这个 Block Table，精准地去不同的物理块中抓取数据进行计算。

3.3 动态按需分配

传统的 LLM 推理是一次性把最大长度的显存要过来。
而在 PagedAttention 中，显存是“按需动态分配”的。每次生成新的 Token，填满了一个 Block，vLLM 才会去显存池里再申请下一个空闲 Block。
这彻底消灭了内部碎片和外部碎片！vLLM 的显存浪费率被压低到了 4% 以下。

3.4 意外之喜：Copy-on-Write (写时复制)

在 OS 中，fork() 子进程时通过页表共享物理内存，只有在修改时才触发 Copy-on-Write。
PagedAttention 也继承了这一神技！在 Parallel Sampling（并行采样） 或 Beam Search 场景下，一个 Prompt 需要生成多个不同的回答。
在以前，同一个 Prompt 的 KV Cache 需要被复制好几份。而在 vLLM 中，多个回答在发散前，它们的 Block Table 完全指向同一个物理 Prompt Block。只有当生成的后续回答开始产生分歧时，才会分配新的 Block。这极大地提升了吞吐量！

4. 结语：计算机科学的轮回

“所有过去皆为序章。”

从 1962 年 Atlas 计算机首次引入分页概念，到 2023 年 PagedAttention 引爆大模型推理速度的革命，底层思想的火花跨越了 60 年，在 GPU 的海量计算单元中再次绽放。

当我们惊叹于 AI 时代的各种“神级架构”时，往往会发现，那些最优雅、最硬核的解法，早就被写在那些沾满灰尘的《操作系统概念》和《计算机体系结构》课本里了。这就是计算机科学的浪漫。

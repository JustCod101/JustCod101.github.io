架构师的魔法：如何让 Claude Code 规划指挥多个 Codex 完成复杂编码任务

在 AI 辅助编程领域，我们正经历一场从“单兵作战”到“兵团协同”的范式转移。

很多人还在把 AI 当作高级的“代码自动补全机器”，但顶级开发者已经开始构建 Manager-Worker 架构的自动化流水线。在这个架构中，Claude Code（或者 Claude 4.6 Opus 等具备顶级逻辑推理与上下文理解能力的模型）扮演“主架构师”（Manager）的角色，而多个轻量级、响应快的代码生成模型（如 Codex、GitHub Copilot CLI 或本地跑的 Llama-3-Coder）则扮演“干活的码农”（Workers）。

本文将深度拆解：如何设计工作流，让 Claude Code 规划并指挥多个 Codex 完成一个真实且复杂的编码任务。

1. 为什么需要“主从架构”？

如果你曾经尝试让一个 AI 一次性写完一个包含前后端、数据库和部署脚本的完整项目，你一定会遇到以下问题：

  1. 上下文遗忘（Lost in the Middle）：写到后端接口时，AI 已经忘了前端需要什么字段。
  2. 逻辑崩塌：一旦在某一步出现 Bug，AI 试图自行修复时往往会陷入“拆东墙补西墙”的死循环。
  3. 输出截断：由于 Token 限制，大项目根本无法在一次对话中输出完毕。

破局之道：分而治之（Divide and Conquer）。
让聪明的脑子（Claude）专心做设计和规划，让干活的手（Codex）去填补具体的代码实现。

2. 核心工作流设计（The Pipeline）

这个流水线的核心在于规范化的文件契约。Claude 不直接写代码，而是写 Markdown 格式的设计文档和任务清单。

Phase 1: 需求消化与架构设计 (Claude Code 的主场)

你向 Claude 输入粗糙的人类需求（如：“写一个基于 React + FastAPI 的在线待办事项，支持拖拽排序”）。
Claude 的任务是生成三份核心文档：

  1. ARCHITECTURE.md：定义技术栈、目录结构、API 接口契约（Swagger/OpenAPI 规范）。
  2. STATE_MACHINE.md：定义前端的状态流转和后端的数据库 Schema。
  3. TASKS.json 或 TASKS.md：将整个项目拆解为多个独立的原子任务（比如：Task 1: 初始化 FastAPI 框架；Task 2: 编写 Todo 的 CRUD 接口；Task 3: 编写 React 拖拽组件）。

Phase 2: 任务派发 (Manager 的分发机制)

Claude 作为一个命令行 Agent（或通过你写的 Python 胶水脚本），开始遍历 TASKS.md。
对于每一个原子任务，Claude 会提取相关的上下文（API 契约 + 该任务的特定描述），组装成一个精准的 Prompt，并非阻塞地发送给下游的 Codex/Copilot 实例。

举个例子：Claude 不会告诉 Codex “去写个后端”，而是告诉它：“阅读这份 API 契约，在 app/routers/todo.py 中实现 POST /todos 接口，要求使用 Pydantic 验证，并配合 SQLAlchemy 写入数据库。”

Phase 3: 编码执行与结果回收 (Codex 的主场)

多个 Codex 实例（或并行调用的 API）接收到高度具象化的 Prompt 后，迅速生成对应的代码文件并落盘。由于任务已经被 Claude 切割得极其细碎，Codex 产生幻觉的概率被降到了最低。

Phase 4: 代码审查与联调 (Claude 的闭环)

代码落盘后，流水线进入 Review 阶段。

  1. 胶水脚本自动运行 pytest 或 npm run test。
  2. 如果报错，将 Error Log 连同报错的源码文件，再次喂给 Claude。
  3. Claude 进行 Root Cause Analysis（根因分析），生成修复指令，再次派发给 Codex，直至测试通过。

4. “上下文隔离”的艺术

在调度多个 Codex 时，最大的忌讳就是把所有项目的代码都丢给它。
最佳实践是：按需提供上下文。
当 Claude 指挥 Codex 写前端组件时，它在 Prompt 中只注入对应的 UI 设计规范和需要调用的单一后端 API 定义，绝不提供数据库建表语句。这种“最小特权原则”（Principle of Least Privilege）不仅省 Token 钱，更能显著提高小模型输出代码的准确率。

5. 展望：AI Native 的软件工程

这种 Claude + 多个 Codex 的协作模式，本质上是在用 AI 模拟一家完整的软件外包公司。

  • Claude = 产品经理 + 架构师 + 测试主管。
  • Codex = 廉价、不知疲倦且高度并行的外包码农。

当我们把这种模式自动化（例如结合 LangChain 或 AutoGen 框架编写调度脚本），我们就不再是在“写代码”，而是在“编写生产代码的流水线”。这是对软件工程的降维打击，也是每一个希望在 AI 时代保持竞争力的开发者必须掌握的魔法。
